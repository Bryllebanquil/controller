ROOT CAUSE: Missing Event Handlers in client.py
================================================================

The Problem:
  - Controller.py sends 'start_stream' event
  - Client.py had NO handler to receive it
  - Result: Streaming never started

The Fix:
  1. Added sio.on('start_stream')(on_start_stream)
  2. Added sio.on('stop_stream')(on_stop_stream)
  3. Created on_start_stream() function
  4. Created on_stop_stream() function

================================================================
WHAT NOW WORKS
================================================================

[OK] Screen Streaming
  - Click "Start Screen Stream" -> Works!
  - Client receives start_stream event
  - Client starts sending screen_frame events
  - UI displays screen

[OK] Camera Streaming
  - Click "Start Camera Stream" -> Works!
  - Client receives start_stream event
  - Client starts sending camera_frame events
  - UI displays camera

[OK] Audio Streaming
  - Click "Start Audio Stream" -> Works!
  - Client receives start_stream event
  - Client starts sending audio_frame events
  - UI plays audio

[OK] Stop Streaming
  - Click "Stop Stream" -> Works!
  - Client receives stop_stream event
  - Client stops sending frame events
  - UI stops displaying stream

START:
